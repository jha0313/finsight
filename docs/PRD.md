# PRD: finsight

## 목표
임의의 카드/은행 명세서 CSV를 업로드하면 Claude가 분석해 **지출 구조·이상 거래·절약 인사이트**를 대시보드로 보여주는 글로벌 핀테크 SaaS.

## 사용자
- **첫 방문자**: 가치·신뢰(보안)·가격을 보고 가입 결정
- **신규 가입자**: 첫 명세서를 올려 빠른 인사이트(Aha)
- **Free 사용자**: 매달 지출을 가볍게 점검(규칙·통계 + Sonnet AI)
- **Pro 구독자**: Opus 심층 분석·절약 인사이트로 행동
- **재방문자(월 주기)**: 새 명세서 추가
- 1차 타겟은 **한국 사용자**(기술 친화적 개인), 글로벌 확장 지향.

## 핵심 기능
### CSV 인제스천 (표준 파서 + Claude 폴백)
- 표준 파서로 먼저 컬럼 매핑·파싱을 시도하고, **인식 실패 시에만** Claude로 컬럼 매핑을 추론 → 사용자 1회 확인/수정 → 이후 결정론적 파싱. 표준 경로는 LLM 비용 0.

### Free (규칙·통계 + Sonnet AI)
1. 카테고리별 지출 분류
2. 기간별 지출 추이
3. 구독 누수·이상 거래 탐지(룰베이스: 반복결제 패턴·이상치)
4. AI 자연어 요약·인사이트(Claude **Sonnet** `claude-sonnet-4-6`)

### Pro (Opus 심층 분석)
5. Claude **Opus**(`claude-opus-4-8`) 심층 분석 — 더 깊은 절약 인사이트·고급 분석. 입력은 카드·계좌번호를 마스킹한 **거래 단위** 데이터(가맹점 수준 구체 인사이트).

- **게이팅**: 서버 DB 구독상태로만 판정. 미구독/쿼터소진 시에도 `/api/analyze`는 **200 + Free 결과 + Pro 잠금 상태**(`pro.status=locked|unavailable`)를 반환.

## 사용자 여정
랜딩(샘플 명세서 데모 먼저) → 구글 로그인 → CSV 업로드 → Free 분석 → Pro 잠금 미리보기 → Polar 결제 → Pro 잠금해제 → 월 주기 재방문.

## MVP 제외 (Post-MVP)
- 국내 PG(토스·카카오페이) 간편결제 — Polar(해외카드/USD)로 시작
- 명세서와의 대화(채팅) — 추후 Pro 기능 후보
- 다국어(i18n) — 한국어 우선으로 시작, 영어는 후속
- 다중 명세서 누적 추세 / 데이터 내보내기
- 수동 컬럼 매핑 오버라이드 UI 고도화 (MVP는 표준 파서 + Claude 폴백 + 1회 확인)
- 다통화 환산 (타 통화 거래는 분리 표시)
- 컬럼 암호화(pgcrypto) — MVP는 식별자 마스킹 + Supabase at-rest로 보호
- 웹훅 out-of-order guard — MVP는 event_id 멱등으로 충분
- 모바일 네이티브 앱 — 반응형 웹으로 충분
- 관측성 지표 파이프라인 / reconciliation cron / rate-limit 인프라(Upstash)
- 계정삭제 UI (cascade FK는 유지)

## 디자인
- 한국어 우선
- 금융 데이터라 **신뢰감 있는 미니멀 UI** + 명확한 보안 메시지(랜딩)
- 차트 중심 대시보드 (카테고리 도넛 + 기간 추이 라인 + 이상거래 하이라이트)
- 반응형 웹 (데스크톱 우선, 모바일 대응)
