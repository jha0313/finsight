핵심 발견 — 어느 게이트에 이빨이 있나

!

의미적 회귀 탐지는 라이브 npm run eval에만 있다

두 번의 break 모두 키 없는 npm test는 여전히 green(26 passed)이었다. 무결성 테스트는 구조·균형(위반4+정상1, id 유일)만 박제할 뿐, "rules.ts의 룰이 케이스와 매칭되는지"나 "CLAUDE.md에 그 사실이 있는지"는 검사하지 않는다 — 설계상 의도된 분리다. 따라서 CI에서 룰/gotcha 삭제를 막으려면 게이트에 npm run eval이 포함돼야 한다.

npm test

이 break들을 통과시킴. 키 불필요·빠름·결정적이지만, 룰/사실 삭제 같은 의미적 회귀는 못 잡는다.

npm run eval

이 break들을 FAIL로 잡음. 라이브 Claude 호출(비용·비결정)이지만 회귀 탐지의 실제 이빨은 여기 있다.
